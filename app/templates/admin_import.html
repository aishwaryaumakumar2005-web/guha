    How Import Works
    • Courses — matched by code. New courses added; duplicates skipped.
    • Students & Tutors — matched by email. New records added; duplicates skipped.
    • Expense Categories — matched by name. New categories added; duplicates skipped.
    • Dependent records (enquiries, fees, attendance, exams, scores, assignments, expenses) — always imported with remapped foreign keys.
    • Users, Payroll, MCQ data — not imported (security & complexity reasons).
    • Source database file is deleted immediately after import.
    • Changes are wrapped in a single transaction — if anything fails, nothing is partially imported.
